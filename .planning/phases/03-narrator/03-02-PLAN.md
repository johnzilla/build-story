**File 1: `packages/core/src/narrate/providers/anthropic.ts`**

Create `AnthropicProvider` class implementing `LLMProvider`:

```typescript
import Anthropic from '@anthropic-ai/sdk'
import { zodOutputFormat } from '@anthropic-ai/sdk/helpers/zod'
import { StoryArcSchema } from '../../types/story.js'
import type { StoryArc } from '../../types/story.js'
import type { Timeline } from '../../types/timeline.js'
import type { FormatType } from '../../types/story.js'
import type { LLMProvider } from './interface.js'
import { buildTimelinePayload } from '../tokens.js'
```

Constructor takes `{ apiKey: string, model?: string }`. Default model: `'claude-sonnet-4-5'`.
Create client: `new Anthropic({ apiKey, maxRetries: 2 })`.

`extractStoryArc(timeline, systemPrompt)`:
1. Build payload via `buildTimelinePayload(timeline)`
2. Call `client.messages.parse()` with:
   - `model` from constructor
   - `max_tokens: 16384` (StoryArc JSON can be large)
   - `temperature: 0` (per NARR-09)
   - `system: systemPrompt`
   - `messages: [{ role: 'user', content: payload }]`
   - `output_config: { format: zodOutputFormat(StoryArcSchema) }`
3. Check `response.parsed_output` is not null (null = refusal). If null, throw Error: "Anthropic refused to generate structured output"
4. Post-validate: `StoryArcSchema.parse(response.parsed_output)` (final safety net per project convention)
5. Return validated StoryArc

`generateFormat(arc, formatType, systemPrompt)`:
1. Serialize `arc.beats` as JSON for the user message
2. Call `client.messages.create()` (NOT .parse — plain text output, per D-04):
   - `model` from constructor
   - `max_tokens: 4096`
   - `temperature: 0`
   - `system: systemPrompt`
   - `messages: [{ role: 'user', content: beatsJson }]`
3. Extract text from `response.content[0]` (type 'text')
4. Return the text string

`synthesizeArcs(arcs, systemPrompt)`:
1. Merge all beats from all arcs into a single JSON array
2. Call `client.messages.parse()` with:
   - `model` from constructor
   - `max_tokens: 16384`
   - `temperature: 0`
   - `system: systemPrompt` (modified to instruct: "Synthesize these beats from multiple chunks into a coherent single StoryArc — reorder chronologically, merge duplicates, ensure narrative flow, preserve all sourceEventIds")
   - `messages: [{ role: 'user', content: mergedBeatsJson }]`
   - `output_config: { format: zodOutputFormat(StoryArcSchema) }`
3. Null-check and post-validate same as extractStoryArc
4. Return validated StoryArc

2. Call `client.chat.completions.create()` (plain text, no structured output):
   - `model` from constructor
   - `temperature: 0`
   - `messages: [{ role: 'system', content: systemPrompt }, { role: 'user', content: beatsJson }]`
3. Extract `completion.choices[0]?.message.content`. If null, throw Error.
4. Return the text string

`synthesizeArcs(arcs, systemPrompt)`:

2. Call `client.chat.completions.parse()` with same structured output pattern as extractStoryArc:
   - `model` from constructor

   - `response_format: zodResponseFormat(StoryArcSchema, 'story_arc')`
3. Same fallback, null-check, and post-validation as extractStoryArc
4. Return validated StoryArc

**File 1: `packages/core/src/narrate/index.ts`** — Replace the stub entirely.

Updated signature to accept an optional LLMProvider, avoiding double instantiation when the CLI already has a provider instance:

`narrate(timeline: Timeline, options: NarrateOptions, provider?: LLMProvider): Promise<StoryArc>`

Also export a `createProvider(options: NarrateOptions): LLMProvider` helper so the CLI can create one provider and pass it to both narrate() and format().

Implementation:
1. Validate `options.apiKey` is non-empty. Throw Error: "API key is required for narration. Set ANTHROPIC_API_KEY or OPENAI_API_KEY environment variable." if empty.
2. Use the injected `provider` parameter if provided. Otherwise, create one via `createProvider(options)`.
3. `createProvider(options)` implementation:
   - `'anthropic'` -> `new AnthropicProvider({ apiKey: options.apiKey })`
   - `'openai'` -> `new OpenAIProvider({ apiKey: options.apiKey })`
   - Anything else -> throw Error: `Unsupported provider: ${options.provider}`
4. Build system prompt via `buildSystemPrompt(options.style, { rootDir: timeline.rootDir, dateRange: timeline.dateRange })`
5. Build payload via `buildTimelinePayload(timeline)`
6. Determine `maxInputTokens` — use `options.maxInputTokens ?? 100000` (D-08 default)
7. **Token guard flow (NARR-08 — correct flow, guard validates chunks not pre-chunked timeline):**
   - Estimate tokens for the full payload via `estimateTokens(payload)`
   - If estimated tokens fit under `maxInputTokens`: call `provider.extractStoryArc(timeline, systemPrompt)` directly
   - If estimated tokens exceed `maxInputTokens`: call `chunkTimeline(timeline, maxInputTokens)` to split into chunks
   - For each chunk: build its payload via `buildTimelinePayload(chunk)`, then call `guardTokens(chunkPayload, maxInputTokens)` — this is where NARR-08 throws. If any individual chunk still exceeds the limit after phase-boundary splitting, `guardTokens` throws with the "estimated X exceeds limit Y" error. This is the legitimate error path: the timeline has a single phase so large it cannot be split further.
   - For each chunk that passes the guard: call `provider.extractStoryArc(chunk, systemPrompt)`
   - After all chunks: call `provider.synthesizeArcs(chunkArcs, systemPrompt)` to merge into a single coherent arc
8. **Post-narration sourceEventIds validation (NARR-05 enforcement):**
   - Collect all valid event IDs from the timeline: `const validIds = new Set(timeline.events.map(e => e.id))`
   - For each beat in the final arc, check that every `sourceEventId` exists in `validIds`
   - If any sourceEventId is not found, log a warning: `Warning: Beat "${beat.title}" references unknown event ID "${badId}" — LLM hallucinated a source link`
   - Do NOT throw on invalid IDs (LLM hallucination is common) — filter them out: `beat.sourceEventIds = beat.sourceEventIds.filter(id => validIds.has(id))`
   - If a beat ends up with zero sourceEventIds after filtering, log a warning but keep the beat
9. Post-validate the final arc with `StoryArcSchema.parse()`
10. Set `metadata.generatedAt` to `new Date().toISOString()`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| core -> LLM API | API keys cross trust boundary; timeline data sent to external service |
| LLM API -> core | Untrusted JSON response from LLM must be validated |
| LLM sourceEventIds | LLM may hallucinate event IDs not present in timeline — must validate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | I (Information Disclosure) | providers/anthropic.ts, openai.ts | mitigate | API keys passed via options (never hardcoded); buildTimelinePayload strips rawContent to minimize data sent to LLM |
| T-03-05 | T (Tampering) | providers/anthropic.ts, openai.ts | mitigate | All LLM responses validated through StoryArcSchema.parse() before returning — untrusted JSON from API cannot bypass Zod validation |
| T-03-06 | D (Denial of Service) | narrate/index.ts | mitigate | Token estimation before API call; chunking splits oversized timelines; guardTokens validates each chunk individually to prevent single oversized request; maxRetries:2 on SDK clients limits retry storms |
| T-03-07 | S (Spoofing) | providers/anthropic.ts, openai.ts | accept | LLM API endpoints are HTTPS with SDK-managed TLS; no additional auth spoofing concern beyond API key theft which is out of scope (env var management) |
| T-03-08 | R (Repudiation) | narrate/index.ts | accept | No audit logging for LLM calls in v1; StoryArc metadata includes generatedAt timestamp for traceability |
| T-03-09 | T (Tampering) | narrate/index.ts sourceEventIds | mitigate | Post-narration validation filters out hallucinated sourceEventIds that don't reference actual timeline event IDs; warnings logged for transparency |
</threat_model>
