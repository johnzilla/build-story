export const BeatTypeSchema = z.enum([
  'idea', 'goal', 'attempt', 'obstacle', 'pivot',
  'side_quest', 'decision', 'result', 'open_loop',

  type: BeatTypeSchema,
  title: z.string(),
  summary: z.string(),
  evidence: z.array(z.string()),
  sourceEventIds: z.array(z.string()),
  significance: z.union([z.literal(1), z.literal(2), z.literal(3)]),

  version: z.literal('1'),

    sourceTimeline: z.string(),
  }),
})

export type BeatType = z.infer<typeof BeatTypeSchema>
export type StoryBeat = z.infer<typeof StoryBeatSchema>

**File 1: `packages/core/src/narrate/providers/interface.ts`**

Create the provider-agnostic LLMProvider interface per D-03:

```typescript
import type { StoryArc } from '../../types/story.js'
import type { Timeline } from '../../types/timeline.js'
import type { FormatType } from '../../types/story.js'

export interface LLMProvider {
  /** Extract a StoryArc from a Timeline using structured output. Per D-01: single call. */
  extractStoryArc(timeline: Timeline, systemPrompt: string): Promise<StoryArc>

  /** Generate formatted text for a given format type. Per D-04: plain text output. */
  generateFormat(arc: StoryArc, formatType: FormatType, systemPrompt: string): Promise<string>

  /** Synthesize multiple StoryArcs (from chunked narration) into a single coherent arc. Per D-06: synthesis call merges chunk arcs. */
  synthesizeArcs(arcs: StoryArc[], systemPrompt: string): Promise<StoryArc>
}
```

**File 2: `packages/core/src/narrate/prompts/system.ts`**

Create 4 style preset system prompts per NARR-03. Each must be meaningfully different. Export a `STYLE_PROMPTS` record keyed by style name and a `buildSystemPrompt(style, timelineMetadata)` function that prepends the StoryArc schema instructions.

The system prompt MUST include:
- Instruction that output JSON must include `version: '1'` (Pitfall 4 from RESEARCH.md)
- The 9 beat types with brief descriptions so the LLM classifies correctly
- `significance` scale: 1=minor, 2=notable, 3=pivotal
- `sourceEventIds` must reference actual event IDs from the input
- `evidence` must quote actual content from the timeline

Style-specific tone differences:
- `technical`: Engineering focus. Emphasize architecture decisions, tradeoffs, technical debt, refactors. Precise language.
- `overview`: High-level product story. Emphasize milestones, features shipped, user impact. Accessible language.
- `retrospective`: Reflective. Emphasize what worked, what failed, pivots, lessons learned. Honest tone.
- `pitch`: Investor/stakeholder framing. Emphasize momentum, velocity, outcomes, vision. Confident tone.

**File 3: `packages/core/src/narrate/prompts/format-prompts.ts`**

Create 4 format-specific system prompts per D-04. Export `FORMAT_PROMPTS` record keyed by FormatType and a `buildFormatPrompt(formatType)` function.

Format specs (from design doc via CONTEXT.md):
- `outline`: Narrative essay, 800-1500 words, full arc with H2 sections per major beat group. Output as markdown.
- `thread`: 8-15 posts, each under 280 characters. Number each post. End with a CTA. Output as markdown with `---` separators between posts.
- `blog`: 500-1000 words, H2 sections, code blocks where relevant, blockquotes for key decisions. Output as markdown.
- `video-script`: 60-120 second narration. Use `[SCENE: beat_type]` markers before each segment. Include timing estimates. Output as markdown.

**File 4: `packages/core/src/narrate/tokens.ts`**

Create token estimation utilities per D-07 and D-08:
- `estimateTokens(text: string): number` — `Math.ceil(text.length / 4)`. Returns 0 for empty string.
- `buildTimelinePayload(timeline: Timeline): string` — Serializes timeline for LLM prompt. MUST omit `rawContent` from events (Pitfall 6 from RESEARCH.md). Include: id, date, source, path, summary, artifactType, crossRefs, metadata. Returns JSON string.
- `guardTokens(payload: string, maxInputTokens: number): void` — Throws Error if `estimateTokens(payload) > maxInputTokens`. Error message must include the estimated count and the limit value.

**File 5: `packages/core/src/narrate/chunker.ts`**

Create phase-boundary chunking per D-06:
- `groupByPhase(events: TimelineEvent[]): Map<string, TimelineEvent[]>` — Extract phase prefix from `event.path` using regex `/phases\/(\d+-[^/]+)/`. Events with no path or no match go to `"ungrouped"` key.
- `chunkTimeline(timeline: Timeline, maxInputTokens: number): Timeline[]` — Uses `buildTimelinePayload` to estimate size. If fits, return `[timeline]`. If not, use `groupByPhase` to split events into groups, then create separate Timeline objects per group (each sharing the original's version/rootDir/scannedAt, with dateRange recomputed from group events). Each chunk must fit under maxInputTokens individually.
