Create `packages/heygen/src/api.ts` implementing:

**Zod response schemas** (validate every HeyGen API response -- never trust raw JSON):
- `HeyGenSubmitResponseSchema`: `{ data: { video_id: string } | null, error: { code: string, message: string } | null }`
- `HeyGenStatusResponseSchema`: `{ data: { video_id: string, status: 'pending' | 'processing' | 'completed' | 'failed', video_url?: string (url), duration?: number, error?: { code: string, message: string } | null } }`

**Error classes:**
- `HeyGenApiError` with `code: string` property. Message format: `HeyGen API error [{code}]: {message}`
- `HeyGenTimeoutError` with `videoId: string` property. Per D-02, message format: `Timeout after {N}s. Video ID: {id} -- check status at https://app.heygen.com/videos/{id}`
- `HeyGenVideoError` with `videoId: string` property. For failed status or stuck detection.

**Helper: `sleep(ms: number)`** -- simple `new Promise(resolve => setTimeout(resolve, ms))`

**`submitChunk(scenes: HeyGenScene[], opts: { apiKey: string; width?: number; height?: number }): Promise<string>`**
- POST to `https://api.heygen.com/v2/video/generate`
- Headers: `X-Api-Key: {apiKey}`, `Content-Type: application/json`
- Body: `{ video_inputs: scenes, dimension: { width: opts.width ?? 1280, height: opts.height ?? 720 } }`
- Wrap the fetch call with `pRetry` (3 retries). Use `shouldRetry` that returns false for `HeyGenApiError` instances (terminal 4xx errors should not retry -- per D-05, Pattern 5).
- Parse response with `HeyGenSubmitResponseSchema.parse(json)`. If `parsed.error` or `!parsed.data`, classify: error code `400140` is daily rate limit (terminal). Any 4xx is terminal. Throw `HeyGenApiError` with the code and message.
- Return `parsed.data.video_id`.

**`fetchVideoStatus(videoId: string, apiKey: string)`** (internal helper)
- GET `https://api.heygen.com/v2/videos/{videoId}` with `X-Api-Key` header.
- Parse with `HeyGenStatusResponseSchema.parse(json)`.
- Return `parsed.data`.

**`pollUntilComplete(videoId: string, opts: { apiKey: string; timeoutSeconds: number }, onProgress: (msg: string) => void): Promise<string>`**
- Per D-01, call `onProgress` with status text on each poll cycle.
- Exponential backoff intervals: `[15_000, 30_000, 60_000, 120_000]` ms -- cap at 120s.
- Track `deadline = Date.now() + opts.timeoutSeconds * 1000`.
- Track `submissionTime = Date.now()` for elapsed display.
- Track `lastStatus` and `lastStatusChangeAt` for stuck detection (10+ minutes at same status = stuck, throw `HeyGenVideoError`).
- On `completed`: return `status.video_url!` (validated by Zod).
- On `failed`: throw `HeyGenVideoError` with `status.error?.message ?? 'Video generation failed'` and `videoId`.
- On timeout: throw `HeyGenTimeoutError` per D-02 format.
- Per Pitfall E, first `onProgress` call after submission should include estimated wait: `Estimated wait: ~{N} minutes (HeyGen averages ~10 min per 1 min of video)`.

**`downloadMp4(videoUrl: string, destPath: string): Promise<void>`**
- Use native `fetch()` to GET the video URL. Per Pitfall B, download IMMEDIATELY from the URL returned by the completed poll -- never cache the URL.
- Stream to disk: `import { createWriteStream } from 'node:fs'`, `import { pipeline } from 'node:stream/promises'`, `import { Readable } from 'node:stream'`.
- `const dest = createWriteStream(destPath)` then `await pipeline(Readable.fromWeb(response.body as ReadableStream), dest)`.
- If `!response.ok || !response.body`, throw `Error('Download failed: HTTP {status}')`.

**`concatMp4s(chunkPaths: string[], outputPath: string): Promise<void>`**
- Per D-03, use FFmpeg concat demuxer via `child_process.spawn` (NOT fluent-ffmpeg -- archived May 2025).
- Build concat list file content: `chunkPaths.map(p => "file '" + p + "'").join('\n')`.
- Write list file to `join(tmpdir(), 'buildstory-concat-{Date.now()}.txt')`.
- FFmpeg binary: `process.env['FFMPEG_PATH'] ?? 'ffmpeg'`.
- Args: `['-f', 'concat', '-safe', '0', '-i', listPath, '-c', 'copy', '-y', outputPath]`.
- Spawn with `stdio: 'pipe'`, resolve on close code 0, reject otherwise.
- Clean up list file after spawn completes: `await unlink(listPath).catch(() => {})`.

**`renderWithHeyGen(arc: StoryArc, config: HeyGenConfig, outputPath: string, onProgress: (msg: string) => void): Promise<HeyGenRenderResult>`**
- This is the PUBLIC function.
- Parse config at boundary: `const opts = HeyGenOptionsSchema.parse(config)` (per adapter.ts validate-at-boundary pattern).
- Call `adaptStoryArc(arc, { avatarId: opts.avatarId, voiceId: opts.voiceId, speed: opts.speed })` to get `AdaptResult`.
- Collect warnings from adapt result into the final result.
- Estimate total minutes for Pitfall E message: sum of beat summaries word count / 150 * 60 / 60 (approximation).
- Sequential chunk processing (per D-03):
  - For each chunk (index `i` of `chunks.length`):
    - `onProgress('Submitting chunk {i+1}/{chunks.length} to HeyGen...')`
    - Call `submitChunk(chunk, { apiKey: opts.apiKey, width: opts.width, height: opts.height })` to get `videoId`.
    - `onProgress('Chunk {i+1}: submitted (video ID: {videoId}). Polling for completion...')`  -- log video_id immediately per anti-pattern rule.
    - Call `pollUntilComplete(videoId, { apiKey: opts.apiKey, timeoutSeconds: opts.timeoutSeconds }, onProgress)` to get `videoUrl`.
    - Download to temp path: `join(tmpdir(), 'chunk-{i}-{videoId}.mp4')` per research recommendation.
    - Call `downloadMp4(videoUrl, tmpPath)`.
    - `onProgress('Chunk {i+1}: downloaded.')`.
    - Track `{ videoId, path: tmpPath }` in `completedChunks` array.
  - If any chunk fails (submission, polling, or download), catch the error. Per D-06: stop processing remaining chunks. Keep downloaded MP4s for successful chunks. Throw an error that includes which chunks succeeded (video IDs and paths) and which failed.
- After all chunks complete:
  - If `chunks.length === 1`: copy/rename the single chunk MP4 to `outputPath` (use `rename` or `copyFile`).
  - If `chunks.length > 1`: call `concatMp4s(chunkPaths, outputPath)`. Per D-04, delete individual chunk MP4 files after successful concat: `for (const p of chunkPaths) await unlink(p).catch(() => {})`.
- Return `{ videoPath: outputPath, warnings: adaptResult.warnings }` typed as `HeyGenRenderResult`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| heygen package -> HeyGen API | Untrusted external API responses cross into the heygen package |
| HeyGen API -> presigned URL | Download URL is an external S3 endpoint |
| CLI -> heygen package | Config/options cross from CLI layer to package |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | api.ts logging | mitigate | Never log apiKey value; only log video_id and status strings |
| T-07-02 | Tampering | api.ts response parsing | mitigate | Zod schema parse on every HeyGen API response (HeyGenSubmitResponseSchema, HeyGenStatusResponseSchema) |
| T-07-03 | Tampering | api.ts file path | mitigate | Use path.join with controlled directory (tmpdir, outputPath from CLI) -- no user input interpolation in paths |
| T-07-04 | Denial of Service | api.ts polling | mitigate | Exponential backoff (15s/30s/60s/120s cap), timeout (600s default), stuck-state detection (10min unchanged) |
| T-07-05 | Spoofing | api.ts presigned URL | accept | URL comes from validated HeyGen API response; downloaded immediately -- no caching that could allow substitution |
</threat_model>
